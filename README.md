📡 Real-Time Ultrasonic Radar SystemA synchronized hardware-software solution that visualizes environmental surroundings through a sonar-style interface. This project utilizes an Arduino Uno to control hardware sensors and Processing 4 to render a dynamic, real-time radar display.🚀 OverviewThe system employs an HC-SR04 ultrasonic sensor mounted on a SG90 servo motor. The servo rotates from $0^\circ$ to $160^\circ$ and back, while the sensor continuously measures distance. This data is streamed via Serial communication to a Processing sketch, which maps the coordinates onto a visual radar sweep.Key Features$160^\circ$ Field of View: Automated sweep for wide-area detection.Visual Proximity Mapping: Detected objects appear as red markers on the digital interface.Hardware Alert: Integrated piezo buzzer triggers a physical alarm when objects enter the "critical zone" (< 25cm).Dynamic UI: Real-time display of angle, distance (cm), and range status.🛠️ Hardware ConfigurationComponentsMicrocontroller: Arduino Uno R3Sensor: HC-SR04 Ultrasonic SensorActuator: SG90 Micro ServoAlert: Active/Passive Piezo BuzzerBreadboard & Jumper WiresWiring MapComponentArduino PinDescriptionUltrasonic TrigPin 8Trigger Pulse OutputUltrasonic EchoPin 9Echo Return InputServo SignalPin 10PWM ControlBuzzer (+)Pin 7Digital Output AlertVCC5VPower SupplyGNDGNDCommon Ground📐 Technical LogicThe distance is calculated based on the speed of sound ($340\text{ m/s}$ or $0.034\text{ cm/}\mu\text{s}$). The formula used in the firmware is:$$distance = \frac{duration \times 0.034}{2}$$The division by 2 accounts for the sound wave traveling to the object and bouncing back.📂 Project StructurePlaintext.

⚙️ Installation & SetupHardware Assembly: Connect the components according to the Wiring Map.Firmware Upload:Open Radar_Firmware.ino in the Arduino IDE.Select your Board and Port.Click Upload.Visualization:Ensure the Serial Monitor in Arduino IDE is closed (to release the COM port).Open Radar_UI.pde in the Processing IDE.Verify the COM port string matches your Arduino port (e.g., "COM3").Click Run.⚠️ TroubleshootingStuck at $0^\circ$: If the radar line doesn't move, press the physical Reset button on the Arduino board while the Processing window is active to re-sync the serial handshake.Port Busy Error: Ensure no other software (like the Arduino Serial Monitor) is using the COM port.Inaccurate Distance: Ensure the sensor has a clear line of sight and is not being powered solely by a low-current USB hub.📄 LicenseThis project is open-source and available under the MIT License.
